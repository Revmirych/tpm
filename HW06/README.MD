Описание задания
Реализация остановки потока через volatile:
Напишите программу, которая использует переменную volatile для безопасной остановки потока.
Реализуйте флаг завершения потока, который будет проверяться в цикле работы потока. Поток должен завершиться, когда этот флаг станет true.
Пример: создайте поток, который выполняет какое-то действие в цикле, пока флаг volatile не будет установлен в true.
 
2. Реализация потокобезопасного счетчика через AtomicInteger:

Создайте класс, использующий AtomicInteger для безопасного увеличения значения счетчика в многопоточном окружении.
Реализуйте методы для инкремента и получения значения счетчика.
Потоки должны параллельно увеличивать счетчик, и программа должна корректно выводить итоговое значение, несмотря на конкуренцию потоков.
 
3. Реализация потокобезопасной структуры данных (singleton-кэш) через AtomicReference:

Реализуйте простую потокобезопасную структуру данных — singleton-кэш, используя AtomicReference.
Кэш должен хранить объект (например, строку) и при обращении к нему возвращать либо существующее значение, либо создавать новое (если текущее значение равно null).
Используйте AtomicReference для обеспечения атомарного обновления значения кэша и синхронизации доступа к нему между потоками.
 
4. Тестирование и производительность:

Проведите нагрузочное тестирование с несколькими потоками, которые будут работать с флагом остановки (volatile), увеличивать счетчик (AtomicInteger) и обращаться к кэшу (AtomicReference).
Оцените, как использование volatile и Atomic-переменных влияет на производительность и синхронизацию в многопоточном контексте.
