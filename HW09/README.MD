Описание задания
1. Использование Future и отмена задач:

создайте пул потоков (ExecutorService)

отправьте в пул долгую задачу (например, имитирующую обработку, длящуюся 5–10 секунд)

получите объект Future для этой задачи

через 2–3 секунды попытайтесь отменить задачу с помощью метода cancel()

 
2. Создание списка из минимум 3–5 задач, каждая из которых:

имитирует выполнение в течение случайного времени (от 1 до 3 секунд)

возвращает результат (например, true/false или число)

используйте метод invokeAll() для одновременного запуска всех задач

 
3. Использование ScheduledExecutorService:

создайте ScheduledExecutorService

настройте периодическую задачу, которая имитирует фоновую операцию (например, «проверка статуса системы», «очистка кэша», «мониторинг очереди»)

 
Вторая часть задания:

Реализуйте сервис PeriodicDataAggregator, который каждые N секунд:

получает список идентификаторов сущностей (например, валют, товаров, городов — в зависимости от варианта);

для каждой сущности асинхронно запрашивает актуальные данные из внешнего источника (имитируется через метод, который может выбрасывать исключения);

обрабатывает успешные результаты (например, логирует или сохраняет).

 
Оформление отчёта:

Составьте отчет, в котором опишите особенности каждого этапа выполнения задания и промежуточные результаты.
Ссылку на отчет прикрепите в личном кабинете на платформе.

 
Чек-лист самопроверки:

создан пул потоков (ExecutorService), в пул отправлена долгая задача, 5-10 секунд по длительности;

получен объект Future для этой задачи;

задача отменяется с помощью метода cancel() через 2-3 секунды;

создан список из 3-5 задач, которые имитируют выполнение в течение случайного времени и возвращают результат;

все задачи запускаются одновременно с помощью метода invokeAll();

создан ScheduledExecutorService;

настроена периодическая задача, которая имитирует фоновую операцию;

реализован сервис PeriodicDataAggregator с указанными условиями;

этапы выполнения задания описаны в отчёте, ссылка на отчёт прикреплена в ЛМС.
