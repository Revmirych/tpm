Домашнее задание по теме 1
Дисциплина: Управление производительностью приложений.

Тема: Метрики производительности: понимание и измерение.

Форма проверки: задание с самопроверкой, это задание преподаватель не проверяет, но мы можем выборочно посмотреть некоторые работ на курсе.

Примерное время выполнения: 3 часа.

Цель задания: практически закрепить навыки работы с инструментами мониторинга и тестирования производительности.

Инструменты для выполнения задания:

Java (любая IDE, например, IntelliJ IDEA или Eclipse);
JDK 8 или выше;
JMeter (для нагрузочного тестирования);
Grafana (для визуализации);
Micrometer/Prometheus (для сбора метрик);
JMH (для микробенчмарков);
VisualVM (для профилирования).
Правила приёма работы:

Выполняйте задание по этапам.
Разместите ссылку на выполненное задание в личный кабинет. Убедитесь, что доступ к документу открыт.
Дедлайн: рекомендуем выполнить задание к следующему вебинару по дисциплине.

 

Описание задания:
Измерьте пропускную способность запросов расчётом метрики через Actuator:

/http.server.requests — выполнить долгий запрос на обработку;
посмотрите значение метрики /http.server.requests count;
через 10 секунд снова сделайте запрос на /http.server.requests count;
вычислите пропускную способность в секунду.
Проведите два замера пропускной способности:

первый замер сделай в методе, где используются обычные (традиционные) потоки;
второй замер сделай в методе, где используются виртуальные потоки;
сравни результаты замеров по пропускной способности.
Подключите Prometheus и Grafana:

настройте Prometheus для сбора метрик приложения;
настройте Grafana для визуализации метрик;
проанализируйте графики по метрикам: Rate, Errors, Duration.
Напишите JMH тест для какого-либо метода проекта. Например, тест на синхронизированную структуру данных (new ConcurrentHashMap<>(); и Collections.synchronizedMap(new HashMap<>()).

Подготовьте краткий отчет:

опишите, какую метрику измеряли и почему она важна;
перечислите инструменты, которые использовали
вставьте скриншоты графиков из Grafana.
Ссылку на отчет прикрепите в личном кабинете на платформе.

 

Чек-лист самопроверки:
Измерена пропускная способность запросов через метрику /http.server.requests с помощью Actuator.
Проведены и выполнено сравнение двух замеров пропускной способности: первый замер — в методе с обычными потоками, второй замер — в методе с виртуальными потоками.
Настроены и подключены инструменты мониторинга: Prometheus — для сбора метрик приложения, Grafana — для визуализации метрик.
Проанализированы графики в Grafana по метрикам: Rate (скорость обработки), Errors (ошибки), Duration (время выполнения).
Написан и выполнен JMH тест для выбранного метода проекта.
Написан краткий отчёт с описанием метрик, инструментов и результатов тестирования.
Ссылка на отчёт прикреплена в личном кабинете.
