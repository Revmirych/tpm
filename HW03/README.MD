Описание задания
Напишите Java-программу, которая создаёт список из 1000000 случайных чисел и выполняет следующие операции над этим списком с использованием обычного stream() и parallelStream():
фильтрация (например, только чётные числа)
преобразование (например, умножение на 2)
агрегация (например, сумма всех чисел)
Измерьте время выполнения каждого варианта и выведите результаты изменения

Проанализируйте: почему один метод работает быстрее другого в конкретной ситуации

Составьте отчёт, в котором приведите:

Описание каждого этапа выполнения задания.
Промежуточные результаты (включая выводы по времени выполнения для обоих вариантов).
Объяснение причин различий в производительности между stream() и parallelStream().
Итоговые выводы о том, когда лучше использовать параллельный поток, а когда последовательный.
Примечания о влиянии параллелизма и параллельности на производительность.
Ссылку на отчёт прикрепите в личном кабинете на платформе.

Выведены результаты времени выполнения и проведён анализ причин различий в скорости выполнения.
