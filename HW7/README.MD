Домашнее задание по теме 7
Дисциплина: Управление производительностью приложений.

Тема: Архитектурные паттерны.

Форма проверки: задание с самопроверкой, это задание преподаватель не проверяет, но мы можем выборочно посмотреть некоторые работ на курсе.

Примерное время выполнения: 3 часа.

Цель задания:

применить Hexagonal Architecture внутри одного Spring Boot-приложения.
Инструменты для выполнения задания:

Java (любая IDE, например, IntelliJ IDEA или Eclipse);
JDK 17 или выше;
Spring Boot (для реализации адаптеров и инфраструктурных слоёв);
JUnit / Mockito (для юнит-тестирования доменного ядра без Spring);
Браузер с DevTools (для проверки работы REST-адаптера);
Зависимость gRPC / protobuf;
Логирование для gRPC.
Правила приёма работы:

Выполняйте задание по этапам.
Разместите ссылку на выполненное задание в личный кабинет. Убедитесь, что доступ к документу открыт.
Дедлайн: рекомендуем выполнить задание к следующему вебинару по дисциплине.

 

Описание задания:
Выделите ядро бизнес-логики (например: «рассчитать среднюю цену», «определить тональность отзыва») — оно должно быть чистым: без @RestController, RestTemplate, JpaRepository и других Spring- или инфраструктурных зависимостей.

Создайте порт (интерфейс) для получения исходных данных (например, DataSource или ReviewFetcher) и порт для вывода результата (например, ResultPublisher).

Реализуйте минимум два адаптера для одного из портов в рамках одного проекта. Например:

для входа: один адаптер читает из файла, другой — из REST API (через RestTemplate).
для выхода: один адаптер сохраняет в базу (JPARepository), другой — возвращает через контроллер (@RestController).
Покажите, что ядро можно протестировать без Spring (например, через юнит-тест с моками адаптеров).
 

Оформление отчёта:
Работа должна позволять оценить:

корректность выделения чистого доменного ядра без инфраструктурных зависимостей;
полноту реализации портов для входа и выхода данных;
различия между адаптерами и их независимость от бизнес-логики;
возможность тестирования ядра без запуска Spring или внешних компонентов;
расширяемость архитектуры за счёт подключения нескольких адаптеров к одному порту.
В результате необходимо предоставить реализованный проект (ядро, порты и адаптеры) и краткий аналитический вывод: как взаимодействуют слои Hexagonal Architecture, насколько легко подменяются адаптеры, корректно ли отделена бизнес-логика, и подтверждается ли возможность независимого тестирования доменного ядра.

Ссылку на отчёт необходимо прикрепить в личном кабинете на платформе.

 

Чек-лист самопроверки:
Выделено чистое доменное ядро без зависимостей от Spring и инфраструктуры.
Созданы входной и выходной порты для работы с данными.
Реализованы минимум два адаптера для одного порта.
Реализованы адаптеры для чтения данных (например, файл и REST API).
Реализованы адаптеры для вывода данных (например, база данных и контроллер).
Архитектура не нарушает зависимость «ядро не знает об адаптерах».
Выполнено юнит-тестирование доменного ядра без Spring (с моками адаптеров).
Подтверждена возможность заменить один адаптер другим без изменения ядра.
Написан краткий отчет с описанием использованных инструментов и результатов.
Ссылка на отчет прикреплена в личном кабинете.
