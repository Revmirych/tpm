Домашнее задание по теме 5
Дисциплина: Управление производительностью приложений.

Тема: Подходы к масштабированию систем.

Форма проверки: задание с самопроверкой, это задание преподаватель не проверяет, но мы можем выборочно посмотреть некоторые работ на курсе.

Примерное время выполнения: 3 часа.

Цель задания:

сравнить REST и WebSocket в клиент-серверном взаимодействии.
Инструменты для выполнения задания:

Java (любая IDE, например, IntelliJ IDEA или Eclipse);
JDK 8 или выше;
JMeter (для нагрузочного тестирования);
Браузер с DevTools.
Правила приёма работы:

Выполняйте задание по этапам.
Разместите ссылку на выполненное задание в личный кабинет. Убедитесь, что доступ к документу открыт.
Дедлайн: рекомендуем выполнить задание к следующему вебинару по дисциплине.

 

Описание задания:
Добавьте к вашему существующему микросервису (парсер вакансий, отзывов и т.п.):
Простой фронтенд на HTML + JavaScript (один файл index.html — без фреймворков)
Два способа получения результата парсинга:
– (REST + polling): фронтенд каждые 500 мс опрашивает /status или /result
– (WebSocket): бэкенд сам отправляет результат через WebSocket, как только парсинг завершён
Замерьте и сравните:
Количество HTTP-запросов за одну операцию (в polling-режиме)
Общий объём переданных данных (можно через DevTools → Network)
Время от запуска до получения результата (в обоих случаях)
Поведение при медленной обработке (например, имитируйте задержку в 5 секунд)
 

Оформление отчёта:
Работа должна позволять оценить:

как работает каждый из двух механизмов получения результата парсинга (REST + polling и WebSocket);
сколько HTTP-запросов формируется при использовании polling;
какой объём данных передаётся в каждом режиме;
за какое время результат доставкивается до фронтенда в обоих вариантах;
как меняется поведение системы при увеличении задержки обработки (искусственные 5 секунд).
В результате необходимо предоставить реализованный проект (фронтенд + оба способа получения результата) и краткий аналитический вывод: как ведёт себя система в каждом режиме, какие показатели были зафиксированы (время, объём данных, число запросов), и какие особенности были выявлены (например, высокая частота запросов при polling).

Ссылку на отчёт необходимо прикрепить в личном кабинете на платформе.

 

Чек-лист самопроверки:
Реализованы два способа получения результата парсинга (REST + polling и WebSocket).
Добавлен простой фронтенд в одном файле index.html.
Выполнена проверка корректности работы обоих режимов.
Замерено количество HTTP-запросов, объём переданных данных и время получения результата.
Проверено поведение микросервиса при добавлении задержки обработки (5 секунд).
Написан краткий отчет с описанием использованных инструментов и результатов тестирования.
Ссылка на отчет прикреплена в личном кабинете.
