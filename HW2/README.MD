Домашнее задание по теме 2
Дисциплина: Управление производительностью приложений.

Тема: Работа сборщика мусора (GC) и управление памятью.

Форма проверки: задание с самопроверкой, это задание преподаватель не проверяет, но мы можем выборочно посмотреть некоторые работ на курсе.

Примерное время выполнения: 3 часа.

Цель задания: отработать включение и анализ логов сборки мусора с созданием искусственной утечки памяти и мониторингом heap-памяти.

Инструменты для выполнения задания:

JVM (Java Virtual Machine);
GC Logging;
jstat;
GCViewer;
VisualVM;
Eclipse Memory Analyzer Tool (MAT);
jmap/jcmd;
IDE (IntelliJ IDEA, Eclipse и др.) для написания и запуска кода;
Google Docs, MS Word или любой редактор для оформления отчёта.
Правила приёма работы:

Выполняйте задание по этапам.
Разместите ссылку на выполненное задание в личный кабинет. Убедитесь, что доступ к документу открыт.
Дедлайн: рекомендуем выполнить задание к следующему вебинару по дисциплине.

 

Описание задания:
Выполняйте задание по шагам:

Включите логирование сборки мусора (GC) при запуске приложения, сохранив логи в файл gc.log.
Создайте искусственную утечку памяти (например, с помощью статической коллекции, в которую постоянно добавляются объекты).
Запустите приложение под нагрузкой (вручную или с помощью простого цикла/эндпоинта), чтобы heap-память заполнилась за 1–2 минуты.
Проанализируйте GC-логи и определите, освобождается ли память после сборки.
Подключите VisualVM или другой профайлер и покажите рост heap-памяти на графике (опционально).
 

Оформление отчёта:
Составьте отчет, который содержит:

пошаговое описание выполнения задания (создание потоков, использование Stream API, симуляция утечек памяти);
описание методов анализа GC-логов и heap dump;
примеры работы с инструментами VisualVM или Eclipse MAT для анализа памяти;
итоговые выводы о производительности, обнаруженных проблемах и способах их устранения;
скриншоты из VisualVM или Eclipse MAT (по желанию).
Ссылку на отчёт прикрепите в личном кабинете на платформе.

 

Чек-лист самопроверки:
Написан исходный код с примерами потоков и утечек памяти.
Включено логирование работы GC с параметрами JVM.
Произведен анализ GC-логов с использованием jstat и GCViewer.
Сгенерирован heap dump с помощью jmap или jcmd.
Проведён анализ heap dump в VisualVM или Eclipse MAT.
Определены проблемы с памятью (например, утечки).
Написан отчёт с описанием найденных проблем и способов их исправления.
