Цель задания

Научиться использовать виртуальные потоки (Virtual Threads) для реализации многозадачных приложений с низкими затратами на создание и управление потоками.
Получить навыки сравнения производительности виртуальных потоков и традиционных потоков (platform threads) при выполнении однотипных задач.

Описание задания
1. Создание задач:

Напишите программу, которая создает 10 000 задач. Каждая задача должна:
Выводить имя своего потока (для этого можно использовать метод Thread.currentThread().getName()).
Имитировать неблокирующую I/O операцию. Это можно сделать с помощью Thread.sleep() или Thread.yield(), чтобы имитировать время ожидания (например, при асинхронных запросах).
2. Использование виртуальных потоков:

Для выполнения задач используйте виртуальные потоки, создавая их с помощью метода Thread.ofVirtual().start(…).
Организуйте выполнение всех 10 000 задач с использованием виртуальных потоков. Виртуальные потоки должны запускаться быстро и использовать меньше ресурсов, чем традиционные платформенные потоки.
3. Использование платформенных потоков (platform threads):

Повторите выполнение тех же задач, но уже с использованием обычных платформенных потоков (классические потоки через new Thread(…).start()).
Сравните производительность и поведение этих двух типов потоков.
4. Сравнение производительности:

Для сравнения производительности измерьте время, необходимое для выполнения всех 10 000 задач с использованием виртуальных потоков и традиционных потоков.
Используйте методы System.nanoTime() или System.currentTimeMillis() для измерения времени выполнения.
5. Использование StructuredTaskScope:

Для управления группой виртуальных потоков используйте StructuredTaskScope (в рамках проекта Loom) для удобного ожидания завершения всех задач.
Обработайте исключения и отмену задач с помощью механизмов, предоставляемых StructuredTaskScope.
6. Тестирование:

Проведите тестирование программы с разными значениями задач и сравните результаты выполнения на виртуальных и платформенных потоках.
Оцените использование ресурсов (CPU, память) и время выполнения.
