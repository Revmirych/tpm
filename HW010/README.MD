Описание задания
 
Первая часть задания:

По списку идентификаторов (например: ID товаров, коды валют, названия городов, номера счетов и т.п.) получите данные из внешнего источника (API, БД, файл, парсер и т.д.).

Для этого реализуйте метод, который:

асинхронно через CompletableFuture получает данные по каждому идентификатору

для каждого результата применяет преобразование (например, форматирование, фильтрация, обогащение)

объединяет два независимых асинхронных результата в один объект (например: цена + курс валюты, название + рейтинг, баланс + ФИО)

возвращает список итоговых объектов, собранных параллельно
Используйте thenCombine, thenApply, thenAccept

Реализуйте метод (название и сигнатуру адаптируйте под ваш проект): public List<FinalResult> fetchAllDataAsync(List<String> ids)

 
Вторая часть задания:

Добавьте обработку ошибок через handle или exceptionally, чтобы сбой одной задачи не ломал всю операцию.

Используйте собственный Executor (не ForkJoinPool.commonPool) для всех асинхронных вызовов.

Создайте пул потоков с именованными демон-потоками и разумным размером.

Применяйте thenApplyAsync(…, executor) для тяжёлых или блокирующих преобразований.

Обеспечьте корректное завершение ExecutorService при остановке приложения.

Добавьте логирование с указанием имени потока и ID запроса для трассировки.

Имитируйте ошибку в одном из источников данных — убедитесь, что метод возвращает частичный результат.

Сравните время выполнения асинхронной и синхронной версий.
